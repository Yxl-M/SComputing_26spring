## 3. Creating an SSH Key Pair

### 3.1 Purpose of the Key Pair

VMs created on CloudVeneto usually allow SSH public-key authentication by default. Therefore, a Key Pair must be prepared before creating the VM.

A Key Pair consists of two parts:

- **Private key:** stored on the local computer and used for SSH login;
- **Public key:** injected into the VM's `authorized_keys` file to authenticate the login.

### 3.2 Creating the Key Pair in the CloudVeneto Dashboard

In the Horizon dashboard, go to:

```text
Compute → Key Pairs
```

Then create a new Key Pair:

| Item | Value |
|---|---|
| Key Pair name | `SC4PS` |

After the Key Pair is generated, the platform displays or provides the private key. This private key must be saved on the local computer.

### 3.3 Saving the Private Key on macOS

Run the following commands in the local terminal:

```bash
mkdir -p ~/.ssh
nano ~/.ssh/SC4PS.pem
```

In the `nano` editor, paste the complete private key text provided by the platform. The private key format is:

```text
-----BEGIN OPENSSH PRIVATE KEY-----
<private key content, not shown in this document>
-----END OPENSSH PRIVATE KEY-----
```

Save and exit `nano`:

1. Press `Ctrl + O` to save;
2. Press `Enter` to confirm the file name;
3. Press `Ctrl + X` to exit.

Then set the private key file permission:

```bash
chmod 600 ~/.ssh/SC4PS.pem
```

> Note: The first attempt to run `chmod` produced the error `No such file or directory` because the private key file had not been saved correctly. After saving the file again with `nano`, the command `chmod 600 ~/.ssh/SC4PS.pem` worked correctly.

## 5. Connecting to the VM from the Local Computer

The final successful connection method used in this homework was SSH `ProxyJump`, specified with the `-J` option. This allows the local computer to connect directly to the VM through the gate jump host.

### 5.1 First Login to the Gate Server and Password Change

Run the following command in the local terminal:

```bash
ssh xyuan@gate.cloudveneto.it
```

The terminal asks for the password:

```text
(xyuan@gate.cloudveneto.it) Password: <enter the initial gate password; input is not displayed>
```

On the first login, the system requires the password to be changed. Follow the prompts:

```text
Current password: <initial gate password>
New password: <new password>
Retype new password: <new password again>
```

After the password is changed successfully, exit the gate server:

```bash
exit
```

### 5.2 Connecting Directly to the VM Through the Gate Server

The general SSH ProxyJump command format is:

```bash
ssh -J <gate_user>@gate.cloudveneto.it -i <private_key_path> <vm_user>@<vm_private_ip>
```

The actual successful command used in this homework was:

```bash
ssh -J xyuan@gate.cloudveneto.it -i ~/.ssh/SC4PS.pem almalinux@10.67.20.70
```

When connecting to the gate server or the VM for the first time, the terminal may show a host authenticity prompt. This is normal:

```text
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

Enter:

```text
yes
```

Then the terminal asks for the gate password because the VM connection goes through the gate jump host:

```text
(xyuan@gate.cloudveneto.it) Password: <enter the updated gate password>
```

After the connection succeeds, the session enters the AlmaLinux VM.

## 6. Installing the C Compilation Environment on the VM

The goal is to install development tools such as `gcc` and `make`, then verify that a C program can be compiled and executed correctly.

### 6.1 Updating the System and Installing Development Tools

Run the following commands inside the VM:

```bash
sudo dnf update -y
sudo dnf groupinstall -y "Development Tools"
```

The `Development Tools` group usually includes basic tools required for C/C++ development, such as `gcc` and `make`.

### 6.2 Verifying That gcc Is Installed

Run:

```bash
gcc --version
```

If the terminal displays gcc version information, the C compiler has been installed successfully.

---

## 7. Writing, Compiling, and Running a C Program

### 7.1 Creating `hello.c`

Run the following command inside the VM to create a test program:

```bash
cat > hello.c <<'EOF'
#include <stdio.h>

int main() {
  printf("Hello CloudVeneto!\n");
  return 0;
}
EOF
```

### 7.2 Compiling the Program

Run:

```bash
gcc hello.c -o hello
```

This command compiles `hello.c` into an executable file named `hello`.

### 7.3 Running the Program

Run:

```bash
./hello
```

Program output:

```text
Hello CloudVeneto!
```

This output confirms that the C compilation environment is correctly configured and that the VM can compile and run C programs.
